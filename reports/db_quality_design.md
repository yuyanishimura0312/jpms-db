# JPMS-DB データ品質向上設計書

## 概要

JPMS-DBが全国785校の私立中学校について「どの学校でも等しく同じ品質のレポートを生成できる」ためのデータ収集スキーマと品質ゲートを設計する文書です。

## 2. 関係者発言収集（7カテゴリ × 3～5件）

### 2.1 校長メッセージ（必須：3件以上）
**取得経路**: 学校HP「校長挨拶」、パンフ、YouTube  
**要件**: 建学の精神への言及、異なる年度、各500-800字

### 2.2 教員インタビュー・コラム（必須：3件以上）
**取得経路**: 学校HP「教員ブログ」、教育メディア  
**要件**: 異なる教科・部活から最低3名、各300-600字

### 2.3 在校生の声（必須：5件以上、学年別）
**取得経路**: 学校HP「生徒インタビュー」、説明会動画  
**要件**: 中学1年・2年・3年から各1名以上、各200-400字

### 2.4 卒業生インタビュー（必須：3件以上、進路別）
**取得経路**: 学校HP「卒業生紹介」、学園動画  
**要件**: 進路別に各1名以上、各300-500字

### 2.5 保護者の声（必須：3件以上）
**取得経路**: 学校HP「保護者メッセージ」、公開ブログ  
**要件**: 現役+OB親バランス、各200-400字

### 2.6 メディア・外部評価（必須：3件以上）
**取得経路**: 中学受験誌、日経・朝日教育面、書籍  
**要件**: 第三者による学校評価、各300-500字

### 2.7 ネガティブ・批判的視点（必須：3件以上）
**取得経路**: 5ちゃんねる、みんなの学校情報「課題」、口コミサイト  
**要件**: 実質的指摘（誹謗除外）、各200-400字  
**重要**: 完全ポジティブ情報は説得力欠如。学校の真の力量理解に必須

### 3.2 関係者発言ゲート

| カテゴリ | 必須件数 | 確認内容 |
|---------|---------|--------|
| 校長メッセージ | 3 | 異なる年度版 |

### 3.3 ゲート判定フロー

```

② 関係者発言 ≥ 25件 → ③ へ、以外 → 「発言不足」
③ メタデータ完全 = 100% → ④ へ、以外 → 「出典未記載」
④ ネガティブ情報 ≥ 6件 → ✅ ゲートOK
   以外 → 「ポジティブ偏向」→ 追加取材指示

ゲートOK → Phase 2 QA進行可
ゲート不合格 → Phase 1.5 補完タスク割り当て
```

## 5. 倫理コンプライアンスチェックリスト

### 5.1 著作権法第32条「引用」要件（全て必須）

- [ ] 引用元が「既に公表」された著作物か
- [ ] 出所明示（学校名・HP URL・取得日）が記載されているか
- [ ] 引用量が「引用元全体の30%以下」か
- [ ] 引用が「補助的」で、オリジナル分析が主体か

### 5.2 個人情報保護法チェック

**禁止**:
- [ ] 学生の顔写真・個人名（HP掲載分は例外）
- [ ] 保護者の実名（ブログ引用は「Aさん（母）」匿名化必須）
- [ ] 教員の個人SNS掘り下げ（個人アカウント禁止）

### 5.3 削除依頼フロー

```
削除依頼受付
  ↓
【判定】著作権侵害 or 個人情報法違反か？
  ├─ YES → 7日以内に当該データ削除
  └─ NO → 理由を書面で説明（削除義務なし）
  ↓
削除実行 or 保留判定 → 削除日・理由を記録
```

## 9. 結論：データ品質設計の3原則

### 原則1：出所の完全な可追跡性（Traceability）
全データについて、「どこから取得したのか」を100%明示し、著作権者・情報源が検証可能にする。

### 原則2：ネガティブ情報の必須化（Balanced Perspective）
完全にポジティブなレポートは信用を失う。各校について「課題」「改善希望」を最低6件含める。

### 原則3：多面的な発言の体系的収集（Multivocality）
校長・教員・在校生・卒業生・保護者・メディア・地域など7つの立場からの発言収集で、学校が立体的に理解される。
