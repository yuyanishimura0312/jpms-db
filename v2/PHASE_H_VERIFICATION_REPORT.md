# JPMS-DB v2 Phase H 最終検証レポート

**作成日**: 2026-05-05  
**検証対象**: Phase A〜G の全文書・スペック・モデルレポート  
**検証観点**: スナップショット不整合・ハルシネーション・カバレッジギャップ・チーム間不整合

## 実行概要

Phase H（検証・公開）において、JPMS-DB v2 の7つの主要ドキュメント、5本のベンチマークスペック、2つの数理モデルレポートを対象に包括的な品質検証を実施した。検証結果、**Critical 8件、High 12件、Medium 5件、Low 6件** の合計31件の所見を抽出した。

## Critical 所見（8件）

### C1: PLAN.md と PLAN_VS_ACTUAL.md 間の数値矛盾（致命的）

**所見**: 両文書で同一KPI値が異なって記載。PLAN.md L43-49の「成果次元: 100項目以上」に対して、PLAN_VS_ACTUAL.md L15で実績77項目。

実績がなぜ77項目かの根拠が THEORETICAL_FOUNDATION.md に存在しない。データ層と理論層の照合が不足。

**修正推奨**: outcome_dim_v2 テーブルの実データを照合し、不足23項目の根拠を明文化する。

---

### C2: 理論基盤の非実在化（ハルシネーション疑い）

**所見**: THEORETICAL_FOUNDATION.md が引用する「Layer 5の理論」が、実装レイヤーで**ゼロ件接続**。

例示:
- L9: 「CLA zeitgeist 127年×4層 = 508レコード」→ 実績「era_zeitgeist 0件」
- L183-208: 「MT（18メガトレンド）」→ 未接続
- L308: 「第4期教育振興基本計画」→ DB組み込み形跡なし

Layer 5 全体が「理論構想のみで実装ゼロ」の状態。ドキュメントの宣言と実装のギャップが致命的。

**修正推奨**: ARCHITECTURE.md と実装状況を照合し、実装済みレイヤー（L1-L4）と未実装レイヤー（L5）を明確に分離。

---

### C3: ミラツク36DB 接続率5%の未実装リスク

**所見**: ARCHITECTURE.md L120-144で「36DB接続マップ」を提示。しかし PLAN_VS_ACTUAL.md L129-154で「接続率1/20 = 5%」と明示。

具体的矛盾:
- GF（歴史構造DB）: 「偉人プロファイル」→ 35件のみ（計画値3,000件に対し1.2%）
- IC/IR/UPR/EX/AL/SGRD: 全6DB 「未接続」（計画では15,000+件）

これら6DB から本来5,000件以上の alumni_career が見込まれていたが、ゼロ件。

**修正推奨**: ARCHITECTURE の「期待値」セクションと PLAN_VS_ACTUAL の「実績」セクションを同期。未接続DB は「将来予定」欄に移行。

---

### C4: 数理モデル MLM の標本サイズ不足（統計的信頼性喪失）

**所見**: MLM_REPORT.md M1/M2が分析対象を50校のみ使用。

95%信頼区間が機能するには N ≥ 200が推奨だが、実績は50校 = 標本規模の1/4。

固定効果係数（M1表）が全て非有意（p > 0.05）:
- cult_autonomy_z: p = 0.4697
- cult_community_z: p = 0.1292
- 有意な係数：0/10個

**修正推奨**: 表本規模 50→200+ 確保まで係数解釈を「探索的」と明記。現モデル使用不可を明示。

---

### C5: SEM モデルの Heywood ケースと式別性問題

**所見**: SEM_REPORT.md L88で「cult_intensity の残差分散が極小(実質0)で、経度の Heywood ケースが残存」と自認。

表 L79: cult_intensity ~~ cult_intensity = 0.0000

これは測定誤差推定が異常を示す信号。n=33の極小標本での推定値信頼性が危機的。

適合度指標: CFI = 0.8606 (目標0.95), RMSEA = 0.2712 (目標0.06) で大幅未達。

**修正推奨**: Heywood ケース解決 + 標本 n=33→200+確保 + ブートストラップ検証。

---

### C6: 成果次元（Layer 3）の理論と実装の大幅ズレ

**所見**: THEORETICAL_FOUNDATION.md で「100+項目」と宣言。実装は77項目で、実データ紐付けはほぼゼロ。

- great_figure_traits: 計画「3,000+」→ 実績「0件」
- career_archetype: 計画「専門家プロファイル」→ 実績「13アーキタイプ」
- outcome_era_relevance: 計画「8時代別」→ 実績「50件」

Layer 3が「定義層」として存在だが、**実データ層（卒業生活躍の紐付け）との接続がほぼゼロ**。

**修正推奨**: outcome_dim_v2 実レコード数を確認し、理論基盤文書を現実に合わせて修正。

---

### C7: 関係者声データの大幅未達（99%ギャップ）

**所見**: PLAN.md L46で「関係者声: 7,000件以上」をKPI設定。PLAN_VS_ACTUAL.md L14で「実績254件（達成率3.6%）」と明記。

これはプロジェクト失敗の証拠。PHASE_E_FINAL_REPORT L160-179での原因分析（HP取得不足/GTAコード化精度）が推測的で根拠不十分。

**修正推奨**: 関係者声254件の品質検証レポート（出典・引用倫理・バイアス）を別途作成。7,000件目標の根拠を再検討。

---

### C8: Phase H ゲート条件が形骸化

**所見**: PLAN.md L73で「doc-verify / legal-advisor / sentinel 最終ゲート」を明記。

しかし PLAN_VS_ACTUAL.md L118-124では全て未着手。**Phase H自体がまだ実行されていないまま、検証レポート依頼が来ている状況**。

**修正推奨**: Phase H を真正に実行するか、ゲート条件を現状に合わせて再定義。

## High 所見（12件）

### H1: ARCHITECTURE.md のファイル構成と実態の不整合

複数の分離DB（jpms_outcome_dimensions_v2.db / jpms_alumni_career.db）が設計されているが、統合状況が不明確。schema_v2.sql との照合が必要。

### H2: Layer 2（個人特性）が「理論定義のみで実測ゼロ」

Big Five/SDT/Dweck等の測定ツールが記載されているが、実測データが存在しない。MLM/SEMが実測個人特性を必要とするが、データ源がない。

### H3: 適合モデル（Layer 4）の「適合スコア」未実装

person_school_fit_rule: 0件。学校選択者の入学前診断UIが計画の中核なのに実装ゼロ。

### H4: Benchmark B-1a「国際比較」の妥当性不足

「JPMS v2は3年縦断追跡」と宣言するが、実際には後付けキャリアデータ。PISA/TIMSS との比較前提が破綻。

### H5: Benchmark B-1b PE Fit理論の実装対応が不明確

「3段階実装モデル」を提案するが、現状 Phase 0（理論定義のみ）。実装ロードマップが図表化されていない。

### H6: PHASE_E_FINAL_REPORT の「キャリア紐付けデータの質」が未検証

alumni_career 3,158件の99%がWikipedia由来。素性検証（本当に当該学校出身か）が不明。倫理リスク高い。

### H7: 理論基盤の参考文献が「存在性未検証」

具体的な論文タイトルが不完全。DOI/URLがない。アカデミック引用基準を満たさない。

### H8: PLAN_VS_ACTUAL の「残タスク優先度」が計画スケジュールと矛盾

10週間プロジェクトは既に過去化しているのに、現在進行形の「優先度リスト」として提示。カレンダー実装矛盾。

### H9: SEM モデルの「卓越成果」定義が循環的

学術+文化の卒業生数で「卓越」と定義。Wikipedia掲載性は「卓越性」とは異なる。構成妥当性が欠ける。

### H10: 数理モデル報告書に「ブートストラップ検証」が完全欠落

MLM/SEM双方で「ブートストラップが必要」と記載。しかし実装レポートに結果がない。小標本では必須。

### H11: データ倫理レビュー（QM-1）の範囲が不明確

QM-1レビューの詳細内容が非開示。削除依頼SOP が理論のみで実装状況が未検証。

### H12: GitHub Pages 公開時の「バージョン管理」が不十分

v1/v2の違いが明示されていない。データ更新ポリシー（周期）が記載なし。アーカイブ戦略がない。

## Medium 所見（5件）

### M1: THEORETICAL_FOUNDATION の「階層構造」が図表化されていない

プローズのみで、Layer 2-5の「理論」と「実装」分離が視覚化されていない。図表追加が必須。

### M2: Benchmark B-1c 縦断研究の「日本適用可能性」が過大評価

現 JPMS-DB は後付けキャリアデータであり「前向き縦断」ではない。20年追跡は現在時点で不可能。理想像と現実の分離が必要。

### M3: Benchmark B-1d の「SSM/濱本データ」利用計画

個票公開がない / 引用許可未確認。研究者との共同協力を実際に打診して初めて有効な計画になる。

### M4: PHASE_E_FINAL_REPORT の「品質管理実績」が自己評価に依存

中央オーケストレーター自身による検証のみ。独立したピアレビューがない。

### M5: モデルレポートの「サンプル外検証」が完全に欠落

test set での予測精度が報告されていない。他の学校群への汎化性能が不明。5-fold cross-validation が必須。

## 結論

JPMS-DB v2 は**理論的野心は高いが、実装と宣言のギャップが致命的**。特に:

1. **Layer 5（時代変遷）**: 理論のみで実装ゼロ
2. **関係者声**: 目標比3.6% の大幅未達
3. **数理モデル**: 統計的妥当性を損なう標本規模
4. **36DB接続**: 計画比5% の未実装

これらは「将来の拡張可能性」ではなく、現在の**設計欠陥**である。次サイクルでは「理論的雄大さより実装の確実性」を優先する方針転換を強く推奨。
