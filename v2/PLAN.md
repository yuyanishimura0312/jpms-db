# JPMS-DB v2 実行計画書

**プロジェクト**: 日本私立中学校包括基盤データベース v2.0
**期間**: 2026-05-05 〜 2026-07-13（10週間）
**承認**: 2026-05-05（依頼者承認済み、Plan Mode 経由）
**最終更新**: 2026-05-05

## 1. 目的

JPMS-DB v1（525校・25成果次元・ヒューリスティック推定）を、以下の四層拡張により「予防的・予測的・縦断的な学校適合インフラ」へ昇格させる:

1. **データ層**: 一次情報の大規模集積（建学理念99→525校、関係者声93件→7,800件）
2. **理論層**: 教育心理学・教育学コア理論30件＋多分野（偉人・経営・起業・研究・人類学・神話）の統合
3. **成果定義層**: 25→100+項目、卒業後5/10/20年の活躍経路を含む多角化
4. **数理モデル層**: MLM/IRT/SEM/LCA/GCM の5種実装

## 2. アーキテクチャ概要

5層構造:
- Layer 1: 学校実態（拡張）
- Layer 2: 個人特性（新設）
- Layer 3: 成果次元（拡張・多角化）
- Layer 4: 適合・予測モデル（新設）
- Layer 5: 時代変遷モデル（新設）

外部接続: ミラツク36DB（FK/PE/CI/GR/CLA/MT/AL/GF/SG/IR/PD/SI/IC/SGPR/SGRD/AA/UPR/AK/MY/AN/SIF/FS/TA/TK/IT/MG/BM/MS/ME/RG/YK/GPT/AD/EX）

詳細は `ARCHITECTURE.md` を参照。

## 4. KPI

- HP一次取得率: 525校中420校以上（80%）
- 関係者声: 7,000件以上
- 成果次元: 100項目以上
- 卒業生活躍データ: 5,000人以上紐付け
- 数理モデル: 5種すべて実装＋論文水準のレポート

## 5. 進捗管理

- 日次ログ: `PROGRESS.md`
- フェーズ完了ゲート: sentinel エージェント承認
- 各フェーズの run-card: `specs/phase_*.md`

## 6. リスク管理

優先順:
1. HP取得の倫理リスク（robots.txt厳守、5秒/req遅延、学校事前連絡）
2. 個人情報・引用倫理（legal-advisor 監修、未成年完全匿名化）
3. データ偏り（地方・独立型に追加リサーチ枠）
4. 数理モデル過信（ヒューリスティック推定との明確な区別）
5. 時間制約（Phase E に人員集中、並列性最大化）
