## 4. 実行計画（4フェーズ）

### Phase 1: 校数カバー 95%達成（48校追加） — 1セッション

**手段**: Wikipedia API + 都道府県別「私立中学校一覧」記事の本文解析

**実行方針**:
- 不足県7つ（広島・京都・岡山・福岡・東京・愛媛・徳島）
- Wikipediaの「Category:○○県の私立中学校」と「○○県の中学校一覧」両方を取得
- 学校紹介記事のリンクから漏れた校を抽出
- 既存と重複チェック後 schools_v2 に追加

**予想収穫**: +30〜40校（広島・京都・岡山が主）

**完了基準**: schools_v2 が 830校以上、文科省比 95%超

### Phase 2: URL紐付け 85%達成（217校追加） — 2セッション

**手段**: 各校の Wikipedia ページ から外部リンクを抽出

**実行方針**:
- 既存 `scripts/url_discovery_30k.py` を URL欠如335校に適用
- Wikipedia API: action=query / prop=extlinks
- 1秒/req の遅延、User-Agent 明示
- `.ed.jp` `.ac.jp` ドメインを優先採用
- パターン:
  1. Wikipedia 記事の外部リンクから公式サイト抽出
  2. 検索エンジン（DuckDuckGo HTML版）で代替
  3. 学校所在地市区町村の私立学校一覧サイト

**倫理規律**:
- robots.txt 厳守
- 1秒/req per domain
- User-Agent: `JPMS-DB-Research/2.0 (+research-contact@miratuku.org)`
- 商用検索APIは使用しない（CSE等の課金API不使用）

**予想収穫**: +180〜220校（成功率 80-95%）

**完了基準**: homepage_url 充填率 85%以上

### Phase 3: HP取得 80%達成 — 1セッション

**手段**: 既存 `scripts/hp_fetch_parallel_30k.py` を新規URL紐付け校に適用

**実行方針**:
- 16並列ワーカー、5秒/req per domain
- max 12pages/校 を取得
- 公開サイトマップ・priority slug（principal/voice/curriculum/about）を優先
- 取得失敗校（404・robots禁止）はログ記録、再試行なし

**完了基準**: raw_html_cache に 640校以上のディレクトリ

### Phase 4: testimonials/culture_score 拡張 — 1-2セッション

**手段**: V8/V9 抽出ロジックの新規HP対象適用

**実行方針**:
- 既存 `/tmp/v8_run.py` `v9_final.py` を新規HP校に適用
- per-school cap: 200件
- role検出 + 教育語フィルタの品質ゲート
- 抽出後、school_culture_score の自動再計算（10次元 × 各校 = 各校10件）

**完了基準**:
- testimonials保有率 70%以上（560校）
- 文化スコア算出率 80%以上（640校）

## 5. メタデータ補完計画（独立フェーズ）

URL/establishment_year/school_corporation/religious_affiliation/gender_type/location_city の6項目を **85%充填** に持ち上げる。

**手段**:
- Wikipedia インフォボックス（{{InfoboxSchool}} テンプレート）から構造化抽出
- HP の「学校概要」「沿革」ページから regex 抽出
- 文科省 学校基本調査 公開CSV からの紐付け

**優先順位**:
1. establishment_year（60.7% → 85%）
2. school_corporation（60.3% → 85%）
3. religious_affiliation（66.2% → 85%）
4. gender_type（66.2% → 85%）
5. location_city（66.2% → 85%）

---

## 6. 倫理・コンプライアンス遵守

全フェーズで以下を厳守:

- **robots.txt 厳守**: `urllib.robotparser` で事前確認
- **遅延設定**: 1秒/req（Wikipedia API）/ 5秒/req per domain（HP取得）
- **User-Agent明示**: `JPMS-DB-Research/2.0 (+research-contact@miratuku.org)`
- **未成年情報**: testimonials 抽出時、生徒・卒業生・保護者は完全匿名化
- **引用倫理**: 30-450字、source_url 100% 紐付け
- **削除依頼SOP**: deletion_request_log で24時間以内対応

## 9. 残るリスクと制約

1. **Wikipedia 未掲載校**: 一部の小規模校は記事自体がなく、URL発見が困難
2. **文部科学省統計の更新ラグ**: 新設校・廃校の反映タイムラグ（最大1年）
3. **HP取得拒否校**: robots.txt で全面禁止する校（文系大学附属に多い）
4. **メタデータの正確性**: Wikipedia インフォボックスの記載漏れ・誤記の可能性

これらは「100%カバー不可能」を前提とし、KPIは現実的な95%/85%/80%/70%設定とした。
