# W08-09: SNS・ブログ・YouTube一次情報経路 スコーピングレポート

**ワーカー**: W08-09  
**タイプ**: Phase 0 スコーピング  
**対象**: 日本私立中学校（JPMS）一次情報入手経路  
**調査日**: 2026年5月  
**品質レベル**: 推定値と確認値を明記

日本の私立中学校について「学校HPには出ない関係者の生の声」を収集する一次情報源の生態系を調査しました。保護者ブログから掲示板まで、複数のプラットフォームを横断的に分析し、Phase 1での収集戦略に必要な情報を整理しています。

---

## 1. 保護者・受験ブログ生態系（主要プラットフォーム別）

### 1.1 アメブロ（Ameba）

**確認値**:
- 公式ジャンル「中学受験」が存在し、検索可能
- 2025年度の投稿件数: 16,109件（年度別タグ）
- ハッシュタグ #中学受験2025 で継続的に投稿が行われている

**プラットフォーム規模**:
- 月間訪問数が多い（約7,500万訪問、2021年データ）
- 読者とのコミュニケーション機能が豊富（コメント、いいね）
- ブログランキング機能で人気記事が可視化

**コンテンツの特徴**:
- SAPIX受講生やその保護者のブログが多く登録
- 塾のテスト結果、模試の成績、志望校選択プロセスなど、親の「日誌」形式
- 親の心理的葛藤（偏差値追求vs学校の多様性等）も記録

**サンプリング戦略上の課題**:
- ランキング上位は優秀なお子さん関連ブログが占める傾向があり、統計的バイアスが存在
- 「見ていただきたい」と思う親が積極的に投稿する自己選択バイアス
- 受験失敗の記録は相対的に少ない可能性

**データ取得方法**:
- RSS フィード形式で取得可能（形式: `http://rssblog.ameba.jp/{ブログID}/rss20.xml`）
- 個別ブログの特定には「ブログID」が必要

### 1.2 はてなブログ

**確認値**:
- ブログプラットフォーム全体の訪問数: 147億訪問（2021年）
- RSS 2.0フォーマット・Atomフォーマット両方で自動配信
- 各ブログの最新30件の記事を配信

**プラットフォーム特性**:
- 検索流入が強い（SEO対応が標準装備）
- コンテンツが資産として長期保存される傾向
- 技術系ユーザーが多く→受験情報も「体系的説明」が多い

**コンテンツの特徴**:
- 「ノウハウ系」の記事が多い（受験戦略、塾選びのポイント等）
- 単なる「日誌」ではなく、分析記事が豊富
- 個人の「失敗から学んだこと」の記録が質高い傾向

**データ取得方法**:
- RSS フィード形式で自動取得可能
- HTMLコード内の `<link rel="alternate" type="application/rss+xml">` で検出可能
- RSS リーダーツールで自動巡回が容易

**テーマ別カバレッジ**:

### 1.4 にほんブログ村（受験ブログ村）

**確認値**:
- 中学受験（本人・親）カテゴリ: **1,336ブログ登録**（最新確認値）
- 大学受験（本人・親）カテゴリ: 1,733ブログ登録
- 年度別のサブカテゴリ存在:
  - 中学受験 2025年度組
  - 中学受験 2024年度組

**重要性**:

**ランキング機能**:
- IN ポイント: リンク元のポイント（にほんブログ村経由の流入）
- OUT ポイント: ブログから他サイトへのリンク
- 人気ランキングで上位ブログが可視化

## 6. プラットフォーム API/取得手法と利用規約

### 6.1 YouTube Data API

**利用可能性**: ✓ 許可あり（規約遵守時）

**方法**:
- Google Cloud プロジェクト登録が必須
- API キーで動画検索・詳細情報取得が可能
- チャンネル統計情報の取得（登録者数、再生数等）も可能

**制限事項**:
- スクレイピングは禁止（「自動化された手段」を明示的に禁止）
- 商用利用に関する制限あり（広告対象に含まない限り無料）
- 利用規約で「透明性」と「ユーザー理解」が必須
- API クォータ制限あり（日単位の利用制限）

**JPMS 用途での適用**:
- 教育用途であっても公式 API 利用が必須
- API キーの取得・申請時に「用途（中学受験情報収集）」を明示必要
- 大規模データ収集の場合は Google への事前申告推奨

### 6.2 X（Twitter）API（2026年版）

**利用可能性**: △ 有料化（2024年以降）

**プラン体系**（2026年版）:
- Free プラン: 廃止（2023年以前は存在）
- Basic プラン: 低額有料（月額 $100 推定）
- Pro プラン: 中規模利用向け（月額 $500 推定）
- Enterprise プラン: 大規模向け（カスタム見積もり）

**機能**:
- ハッシュタグ検索: 利用可能
- ツイート履歴検索: 利用可能
- 検索精度: キーワード・ハッシュタグ両対応
- リアルタイム検索: 可能

**JPMS での課題**:
- 無料利用廃止により、直接的なコスト発生
- 学術目的での割引制度: 確認値なし（事前申請で割引の可能性あり）
- 大規模収集の場合、月額費用が顕著

### 6.3 Instagram Graph API

**利用可能性**: △ 制限あり（2016年以降の厳格化）

**制限内容**:
- ハッシュタグ検索: ビジネスアカウント認証が必須（個人アカウントは不可）
- Sandbox モード制限:
  - 10人のユーザーの最新20件投稿のみ取得可能
  - 1時間あたり500回の API 呼び出し制限
  - 7日ローリング期間で最大30ハッシュタグの監視上限

**データ取得の課題**:
- アカウント名が取得できない（プライバシー保護設計）
- 大規模サンプリングには不向き
- ビジネスアカウント認証プロセスが時間かかる（却下リスクあり）

**推奨**: Instagram は手動で主要アカウント（学校公式等）を抽出する方が実用的

### 6.4 Ameba・はてなブログ RSS

**利用可能性**: ✓ 許可（RSS標準）

**取得方法**:
- **アメブロ**: `http://rssblog.ameba.jp/{ブログID}/rss20.xml`
- **はてなブログ**: `https://{blogname}.hatenablog.com/rss`

**利用規約での扱い**:
- RSS 標準なので、利用は許可されている傾向
- サーバー負荷防止のため「礼儀的な」制限あり（過度な高頻度アクセスは避ける）
- 個別ブログの特定には「ブログID」が必要（マッピング作業が必要）

**実装上の考慮**:
- RSS リーダーツール（Python の feedparser 等）で自動巡回が容易
- 定期取得（日次または週次）で十分

### 6.5 note API

**利用可能性**: △ 限定的

**現状**:
- 公開 API なし（note 社から公式 API が提供されていない）
- 記事へのアクセスは HTML スクレイピング必須（BeautifulSoup 等を使用）
- タグ検索: Web UI でのみ確認可能（note.com/hashtag/〇〇）

**法的リスク**:
- 利用規約で HTML スクレイピングを明示的に禁止していない傾向
- しかし、note 利用規約の「自動取得禁止」条項に抵触する可能性がある
- 大規模スクレイピングの場合は note 社への事前確認推奨

## 7. 倫理的引用ガイドライン（Phase 1向け一次案）

### 7.1 日本著作権法に基づく引用の要件

**必須要件（すべて満たす必要）**:
1. **公表性**: 公開されている著作物であること（SNS投稿、ブログ記事等）
2. **引用の必然性**: 報道、批評、研究の目的のためなど、引用を行う「必然性」があること
3. **正当な範囲内**: 引用の「正当な範囲内」であること（記事全体の10%程度が目安）
4. **主従関係**: 引用部分とそれ以外の部分の「主従関係」が明確（引用部分が従）
5. **明瞭な区分**: カギ括弧などにより引用部分と自分の著作物とが「明瞭に区分」されていること
6. **改変禁止**: 引用する他人の著作物を改変していないこと
7. **出所の明示**: 「出所が明示」されていること（ブログ名・執筆者・URL等）

### 7.2 個人ブログの取り扱い

**注意事項**:
- 個人保護者ブログであっても、著作物として法律上保護される
- 「見守り目的」の観察だけでは引用の「必然性」がない（著作権侵害となる可能性）
- 私的利用の範囲を超える（ブログ村等での公開は私的利用ではない）

**推奨される取り扱い**:
- 個別インタビューを実施する（明示的な許諾取得）
- 統計的集約に限定（個別ブログの直接引用は避ける）
- 例: 「ブログ村に登録されている1,336ブログの分析から、親の関心は…」
- 事前に執筆者への利用許諾を取得（メール等で依頼）

### 7.3 SNS・掲示板（5ch等）の取り扱い

**5ch スレッド**:
- **原則として引用対象外にする**（匿名性が高く、著作権者の特定が困難）
- 学校評価等の「統計的パターン」を別途分析する場合は、個人特定が不可能な形式で集約
- 例: 「5ch での開成学園スレッドでは、入試難易度に関する投稿が多い」→ OK
- 例: 「○○さんの投稿によると…」→ NG

**X（Twitter）**:
- **公開アカウント（フォロワー数多い）は引用可能**
- 個人垢のリツイートは原則引用対象外
- ハッシュタグまとめは「統計利用」として明記
- 引用時には URL とアカウント名を明示

### 7.4 個人情報（PII）の取り扱い

**非開示基準**:
- 生徒名・学年・入試成績・合格学校名は匿名化
- 親の職業・学歴等は削除
- 進学先の詳細情報も非開示
- 塾名や家庭教師の指名も慎重に（個人の学習プロセスに特定される可能性）

**匿名化の方針**:
- 「SAPIX に通う小5の保護者」→ OK（一般的な属性）
- 「〇〇中学に合格した○○君の母」→ NG（生徒が特定される）
- 「小4の2月から通塾を開始した家庭」→ OK

### 7.5 ブログプラットフォーム別の倫理基準

| プラットフォーム | 著作権 | PII 取扱難度 | 推奨方法 |
|---|---|---|---|
| **個人ブログ（アメブロ等）** | 厳格 | 慎重 | 許諾取得またはインタビュー |
| **note** | 厳格 | 慎重 | 公開記事は引用可（出所明示） |
| **5ch** | 緩い | **最注意** | 統計集約のみ、個別引用は避ける |
| **X** | 中程度 | 中程度 | フォロワー多のアカウントのみ |
| **Instagram** | 厳格 | **最注意** | 学校公式アカウントのみ |

## 8. Phase 1 への提言：優先順位付け収集戦略

### 8.1 優先度の判定基準

| 基準 | 評価軸 |
|---|---|
| **一次情報性** | 学校HPに出ない「生の声」か |
| **規模（サンプル数）** | 十分な件数があるか |
| **確実性** | データ取得方法が確立しているか |
| **倫理的リスク** | 個人情報・著作権リスクは低いか |
| **費用** | API 有料化等のコスト |
| **実装複雑度** | パイプライン構築の難易度 |

### 8.2 推奨される優先順位付け

#### **Tier 1（最優先）: 即座に開始**

**1. にほんブログ村「受験ブログ村」**
- 理由: 1,336ブログが登録済み、集約ハブとして機能、サンプリングの「入口」として最適
- 方法: 各ブログへのリンク取得 → 個別 RSS フィード自動収集
- コスト: 無料
- 倫理的リスク: 低（公開ブログ、RSS 許可想定）
- 実装: Python RSS リーダー + 定期自動取得（日次）
- 期待成果: 67,000件（1,336ブログ × 最新50記事）

**2. note の中学受験タグ**
- 理由: 57,598件の公開記事、プロ発信者が多い、一次情報の質が高い
- 方法: タグページの定期スクレイピング（HTMLパース）
- コスト: 無料
- 倫理的リスク: 中程度（出所明示で対応）
- 実装: Selenium/BeautifulSoup での月次取得（新記事追跡用）
- 期待成果: 57,598件の初回スナップショット + 月次更新

**3. X（Twitter）ハッシュタグ収集**
- 理由: リアルタイム性が高い、学校別情報が豊富、親の「今の声」が記録される
- 方法: X API（有料）でハッシュタグ検索（#中学受験, #〇〇中学等）
- コスト: 月額 $100~500 推定（Basic プラン）
- 倫理的リスク: 中程度（公開アカウントに限定、個人垢は除外）
- 実装: X API v2 + BigQuery 統合（推奨、後処理効率化）
- 期待成果: 日単位で1,000件以上の X ツイート蓄積

#### **Tier 2（次フェーズ開始）: Phase 1 後半**

**4. YouTube チャンネル統計**
- 理由: 学校紹介・受験情報の動画化が進行中、学校の「新しい発信方法」の把握
- 方法: YouTube Data API でチャンネル情報取得
- コスト: API 無料枠あり（月額 10,000 単位まで無料）
- 倫理的リスク: 低
- 実装: チャンネルID リスト化（コベツバ等20~50件） → 定期取得

**5. アメブロ 中学受験タグ**
- 理由: 16,109件の年次投稿、親の「声」が最も直接的
- 方法: RSS フィード + タグ別スクレイピング（新着投稿自動取得）
- コスト: 無料
- 倫理的リスク: 高（個人ブログが多い、親の特定情報が含まれる）
- 実装: 「統計的集約」に限定、個別ブログの引用は許諾取得が必須
- 期待成果: 統計的パターン分析（個別ブログの引用ではなく）

#### **Tier 3（Phase 2以降）: 詳細分析向け**

**6. 5ch 「お受験」板**
- 理由: 学校別スレッドで率直な評価、偏差値・難易度の詳細情報が豊富
- 方法: スレッドのテキスト取得 + 自然言語処理（センチメント分析等）
- コスト: 無料
- 倫理的リスク: 最高（匿名ユーザー、未確認情報、誹謗中傷の可能性）
- 実装: 「学校評価のセンチメント分析」用途に限定、個人情報は完全削除

**7. Instagram 学校公式アカウント**
- 理由: 学校の「自発的情報発信」、新しい広報戦略の把握
- 方法: 学校リストを作成 → 手動または Graph API（Sandbox）
- コスト: 無料（Sandbox） または API 認証（有料）
- 倫理的リスク: 低
- 実装: Web UI での手動サンプリング（数十校パイロット）

**8. 帰国子女コミュニティ**
- 理由: 専門的な学校情報（英語選択入試等）、多様性確保
- 方法: note/ブログ + InterEdu 掲示板の手動収集
- コスト: 無料
- 倫理的リスク: 中程度
- 実装: 帰国子女向けタグ別に note で検索、InterEdu掲示板の月次スクリーニング

## 9. 留意事項と制約

### 9.1 推定値が多い領域

以下の領域は、確認値が限定的なため、Phase 1 での実装時に改めて精査が必要です。

- YouTube 各チャンネルの具体的な登録者数・再生数（公開データソースの検索性問題により、チャンネル直接アクセスが必須）
- 私立中学校公式 YouTube チャンネルの普及度（「数十校」～「数百校」の幅広い推定）
- Instagram 学校公式アカウントの数（確認値なし、学校リストとのマッピング必要）
- X 上の中学受験ツイート件数（リアルタイム API アクセス必要、日単位で変動）

### 9.2 API 利用費用の不確実性

- **X API**: 2026年時点で新しいプラン体系。学術割引有無は未確認（Google の Academic Research program 的な割引制度の有無を確認必要）
- **Instagram Graph API**: ビジネスアカウント認証が必須（初回申請に時間必要、却下リスク存在）
- **YouTube Data API**: 無料枠で足りるか、大規模収集時の追加コスト不明。予算見積もり時に慎重に見積もり必要

### 9.3 倫理・法的リスクの不完全性

- **個人ブログ引用**: 「必然性」判断は案件ごとに異なる可能性（法的確実性なし、案件ごとの弁護士相談推奨）
- **5ch 掲示板**: テキスト利用の法的扱いが曖昧（判例少ない）
- **著作権法の改正**: 2024年以降の改正により、AI学習用途でのテキスト利用制限が予定されている可能性
